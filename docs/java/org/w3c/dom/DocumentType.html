Interface DocumentType

  • All Superinterfaces:
    Node 

    public interface DocumentType
extends Node
    每个Document都有一个doctype属性，其值为null或DocumentType对象。 该DocumentType在DOM核心接口提供到为文档定义，并没有别的，因为命名空间和DTD表示各种XML模式的努力的效果都没有明确理解为写这篇文章的实体列表的接口。

    DOM Level 3不支持编辑DocumentType节点。 DocumentType节点是只读的。

    另见Document Object Model (DOM) Level 3 Core Specification 。

      • getEntities

        NamedNodeMap getEntities()
        A NamedNodeMap包含在DTD中声明的外部和内部的一般实体。 不包含参数实体。 复制件被丢弃。 例如： 
          <!DOCTYPE
 ex SYSTEM "ex.dtd" [ <!ENTITY foo "foo"> <!ENTITY bar
 "bar"> <!ENTITY bar "bar2"> <!ENTITY % baz "baz">
 ]> <ex/>
        该界面提供访问foo和第一个声明bar但不是第二个声明bar或baz 。 该地图中的每个节点也实现了Entity接口。
        DOM Level 2不支持编辑实体，因此entities不能以任何方式进行更改。

      • getNotations

        NamedNodeMap getNotations()
        A NamedNodeMap包含在DTD中声明的符号。 复制件被丢弃。 该地图中的每个节点也实现了Notation接口。
        DOM Level 2不支持编辑符号，因此notations不能以任何方式进行更改。
