Interface DOMImplementationSource


  • public interface DOMImplementationSource
    该接口允许DOM实现者根据请求的特征和版本提供一个或多个实现，如DOM Features 所述 。 每个实现的DOMImplementationSource对象都列在可用源的绑定特定列表中，以使其DOMImplementation对象可用。

    另见Document Object Model (DOM) Level 3 Core Specification 。

    从以下版本开始：
    DOM 3级

    • 方法详细信息

      • getDOMImplementation

        DOMImplementation getDOMImplementation(String features)
        一种请求支持指定功能的第一个DOM实现的方法。
        参数
        features - 指定需要哪些功能和版本的字符串。 这是一个空格分隔的列表，其中每个要素由其名称指定，后跟空格和版本号。 此方法返回getDOMImplementationList返回的列表的第一个项目。 作为示例，字符串"XML 3.0 Traversal +Events 2.0"将请求支持其3.0版本的模块“XML”的DOM实现，该模块支持任何版本的“Traversal”模块，以及2.0版本模块“Events”。 必须使用方法Node.getFeature()和DOMImplementation.getFeature()模块“事件”。
        结果
        第一个支持所需功能的DOM实现，如果此源没有， null 。
