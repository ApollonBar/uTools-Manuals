      • loadClass

        public static 类 loadClass(String className,
                              String remoteCodebase,
                              ClassLoader loader)
                       throws ClassNotFoundException
        返回指定类的类实例。

        该方法的规范是“Java to IDL语言映射”ptc / 00-01-06。

        在Java SE平台中，此方法的工作原理如下：

        • 查找调用堆栈上的第一个非空值ClassLoader ，并尝试使用此ClassLoader加载该类。
        • 如果第一步失败，如果remoteCodebase不为空，并且useCodebaseOnly为false，则调用java.rmi.server.RMIClassLoader.loadClass(remoteCodebase, className) 。
        • 如果remoteCodebase为null或useCodebaseOnly为true，则调用java.rmi.server.RMIClassLoader.loadClass(className) 。
        • 如果一个课程没有成功加载步骤1,2或3，并且loader是非空的，那么请致电loader.loadClass(className) 。
        • 如果一个类成功地由步骤1,2,3或4加载，则返回加载的类，否则抛出ClassNotFoundException 。
        参数
        className - 类的名称。
        remoteCodebase - 可以找到类的空格分隔的URL列表。 可能为null。
        loader - 一个 ClassLoader ，如果所有其他方法都失败，可以用于加载类。
        结果
        表示加载类的 类对象。
        异常
        ClassNotFoundException - 如果类无法加载。

      • isLocal

        public static boolean isLocal(Stub stub)
                       throws RemoteException
        isLocal方法具有与ObjectImpl._is_local方法相同的语义 ，除了它可以抛出一个RemoteException 。 提供_is_local()方法，使得存根可以确定特定对象是否由本地服务器实现，因此可以使用本地调用API。
        参数
        stub - 要测试的存根。
        结果
        _is_local()方法返回true，如果表示对象的servant位于与存根相同的进程中，并且它们共享相同的ORB实例。 _is_local()方法返回false。 _is_local()的默认行为是返回false。
        异常
        RemoteException - Java到IDL规范没有指定引起 RemoteException的条件 。
