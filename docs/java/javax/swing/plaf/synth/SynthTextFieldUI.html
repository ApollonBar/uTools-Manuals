    为JTextField提供Synth L＆F UI代表。

    警告：此类的序列化对象与将来的Swing版本不兼容。 当前的序列化支持适用于运行相同版本的Swing的应用程序之间的短期存储或RMI。 从1.4开始，对所有JavaBeans的长期存储的支持已被添加到java.beans包中。 请参阅XMLEncoder 。

    1.7

      • paint

        protected void paint(SynthContext context,

        这将被传递到BasicTextUI.paintSafely(java.awt.Graphics)方法，保证模型在渲染时不会从此线程的视图中更改（如果相关模型来自AbstractDocument ）。 这样可以使模型有可能异步更新。

        protected void propertyChange(PropertyChangeEvent evt)
        当在关联的JTextComponent上更改bound属性时，将调用此方法。 这是一个钩子，UI实现可能会更改，以反映UI如何显示JTextComponent子类的绑定属性。 这被实现为不做任何事情（即，在调用此方法之前处理JTextComponent中对属性的响应）。
