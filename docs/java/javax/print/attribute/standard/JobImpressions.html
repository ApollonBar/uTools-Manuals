    JobImpress类是一个整数值打印属性类，它指定要提交的文档的展示次数的总大小。 “印象”是施加到单个媒体页面上的图像（可能是不同配置中的许多打印流页面）。

    JobImpressions属性描述作业的大小。 这个属性不是一个计数器; 如果已知的话，它是有用的路由和调度信息。 如果打印请求中未提供JobImpressions属性的值，打印机可能会尝试计算。 即使客户端确实为打印请求中的JobImpressions属性提供了一个值，如果打印机能够计算比客户端提供的值更准确的值，打印机也可以选择更改该值。 打印机可能能够在作业提交时间或任何后来的时间点确定JobImpressions属性的正确值。

    与JobKOctets一样 ，JobImpressions值不能包括由Copies属性指定的份数所贡献的乘数，而与设备是否可以处理多个副本而不进行多次通过作业或文档数据的传递无关，而与输出无关是否整理。 因此，该值独立于实现，并反映了与次数无关的以印象为单位的文档的大小。

    与JobKOctets一样 ，由于在文档数据中嵌入的副本指令 ，JobImpressions值也不能包含乘数因子。 如果文档数据实际上包含文档数据的复制，则此值将包括此类复制。 换句话说，此值始终是源文档数据中的展示次数，而不是作业生成的展示次数的度量。

    IPP兼容性：整数值给出IPP整数值。 由getName()返回的类别名称提供IPP属性名称。
