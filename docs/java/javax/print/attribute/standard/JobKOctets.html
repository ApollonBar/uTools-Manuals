    JobKOctets类是一个整数值打印属性类，用于指定以K个八位字节为单位的文档的总大小，即以作业中请求处理的1024个八位字节为单位。 值必须向上舍入，所以1到1024个八位字节之间的作业必须表示为1K个八位字节，1025到2048必须是2K字节等。对于多点打印作业（具有多个文档的作业），JobKOctets值通过将单个文档的大小以八位字节相加，然后舍入到下一个K个八位字节值来计算。

    JobKOctets属性描述作业的大小。 这个属性不是一个计数器; 如果已知的话，它是有用的路由和调度信息。 如果打印请求中未提供JobKOctets属性的值，打印机可能会尝试计算。 即使客户端确实为打印请求中的JobKOctets属性提供了一个值，如果打印机能够计算比客户端提供的值更准确的值，打印机也可以选择更改该值。 打印机可能能够在作业提交时间或任何稍后的时间点确定JobKOctets属性的正确值。

    JobKOctets值不能包括由Copies属性指定的份数所贡献的乘数，而不管设备是否可以处理多个副本，而不会在作业或文档数据上进行多次传递，而与输出是否整合不一样。 因此，该值与实现无关，并且指示与K个八位字节中独立于副本数量测量的文档的大小。

    JobKOctets值也不得包含乘法因子，因为嵌入在文档数据中的拷贝指令。 如果文档数据实际上包含文档数据的复制，则此值将包括此类复制。 换句话说，此值始终是源文档数据的大小，而不是要生成的硬拷贝输出的度量。
