    public class AuthenticationNotSupportedException
extends NamingSecurityException
    当不支持所请求的身份验证的特定风格时，会抛出此异常。 例如，如果程序尝试使用强身份验证，但目录/命名仅支持简单身份验证，则会抛出此异常。 识别身份验证的特定风格是提供者和服务器特定的。 可以使用诸如使用SASL标识的特定认证方案或通用认证说明符（诸如“简单”和“强”）来指定它。

    如果程序想要特别处理这个异常，那么在尝试捕获NamingException之前，它应该明确地捕获AuthenticationNotSupportedException。 捕获AuthenticationNotSupportedException后，程序可以通过相应地更新解析的上下文的环境属性来重新尝试使用不同身份验证的身份验证。

    适用于NamingException的同步和序列化问题直接在这里应用。
