    访问控制列表（ACL）中的条目。

    此类表示的ACL条目基于RFC 3530: Network File System (NFS) version 4 Protocol中指定的ACL模型。 每个条目有四个组件如下：

    1. type组件确定条目是否准予或拒绝访问。

    2. principal组件，有时称为“谁”组件，是一个UserPrincipal对应于该条目授予或拒绝访问的身份

    3. permissions组件是一组permissions

    4. flags组件是一组flags ，用于指示条目如何继承和传播

    使用关联的AclEntry.Builder对象通过调用其build方法创建ACL条目。

    ACL条目是不可变的，并且可以安全地被多个并发线程使用。

    从以下版本开始：
    1.7
