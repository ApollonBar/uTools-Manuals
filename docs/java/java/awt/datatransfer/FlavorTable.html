    public interface FlavorTable
extends FlavorMap
    一个FlavorMap，放宽了地图传统的1对1限制。 允许将风味映射到任何数量的本地人，同样，本地人也可以映射到任何数量的口味。 FlavorTables不必是对称的，但通常是。

      • getNativesForFlavor

        List<String> getNativesForFlavor(DataFlavor flav)
        返回List的String当地人到指定的DataFlavor对应。 List将从最好的本地到最坏的排序。 也就是说，第一个本机将最好将指定风味的数据反映到底层本机平台。 返回的List是这个FlavorTable的内部数据的可修改副本。 客户端代码可以免费修改List而不影响该对象。
        参数
        flav -在DataFlavor ，其相应的当地人应该返回。 如果null指定，目前已知这一切当地人FlavorTable在一个不确定的顺序返回。
        结果
        java.util.List的 java.lang.String对象，它们是平台特定数据格式的平台特定表示

      • getFlavorsForNative

        List<DataFlavor> getFlavorsForNative(String nat)
        返回List的DataFlavor s到其指定的String对应。 List将从最好的DataFlavor排序到最差。 也就是说，第一个DataFlavor将最好地反映指定的本机到Java应用程序的数据。 返回的List是这个FlavorTable的内部数据的可修改副本。 客户端代码可以自由修改List而不影响此对象。
        参数
        nat - 应该返回对应的DataFlavor的本机。 如果null null，则此FlavorTable当前已知的所有DataFlavor都以非确定性顺序返回。
        结果
        java.util.List的 DataFlavor对象，指定的平台特定的本机的平台特定数据可以被翻译成
