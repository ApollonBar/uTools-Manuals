      • SOFT_FAIL

        public static final PKIXRevocationChecker.Option SOFT_FAIL
        如果撤销状态由于以下原因之一而无法确定，允许撤销检查成功：
        • 由于网络错误，无法获取CRL或OCSP响应。
        • OCSP响应者返回RFC 2560第2.3节中指定的以下错误之一：internalError或tryLater。

        请注意，这些条件适用于OCSP和CRL，除非设置了NO_FALLBACK选项，否则撤销检查只有在两个机制在上述条件之一失败时才能成功。 导致网络错误的异常被忽略，但可以稍后通过调用getSoftFailExceptions方法检索 。
