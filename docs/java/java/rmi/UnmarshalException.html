    public class UnmarshalException
extends RemoteException
    一个UnmarshalException可以同时如果任何下列条件发生解组参数或远程方法调用的结果被抛出：
    • 如果在解组调用头部时发生异常
    • 如果返回值的协议无效
    • 如果java.io.IOException发生解组参数（在服务器端）或返回值（在客户端）。
    • 如果一个java.lang.ClassNotFoundException期间解组参数或返回值发生
    • 如果没有骨架可以在服务器端加载; 请注意，在1.1 stub协议中需要骨架，但不在1.2 stub协议中。
    • 如果方法哈希无效（即，缺少方法）。
    • 如果在未组态远程对象的存根时，如果无法创建远程引用对象。
    从以下版本开始：
    JDK1.1
